<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> `/api/internal/sync-wise` | Authenticated admin browser can trigger an expensive Wise sync. |
| Vercel Cron -> `/api/internal/sync-wise` | Cron GET remains bearer-secret authenticated. |
| Sync route -> Wise API | Trigger contacts Wise heavily and may run close to 300s. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.7-09 | Spoofing | Manual sync POST | mitigate | Require `auth()` session when cron secret is absent/invalid; `/api/internal/*` bypasses middleware so route self-authenticates. |
| T-08.7-10 | Information Disclosure | Browser manual sync | mitigate | Browser sends no `CRON_SECRET`; tests/grep assert no secret/header reference in client file. |
| T-08.7-11 | Denial of Service | Sync button | mitigate | Confirmation prompt, disabled running state, and duplicate-click prevention. |
| T-08.7-12 | Tampering | Cache invalidation | mitigate | `revalidateTag("snapshot", { expire: 0 })` remains server-route only and success-gated. |
</threat_model>
