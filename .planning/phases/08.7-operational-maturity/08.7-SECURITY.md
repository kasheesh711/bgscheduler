# Phase 08.7 - Security

Per-phase security contract for Phase 08.7 operational maturity. This audit verifies the STRIDE threat registers declared in the five Phase 08.7 PLAN files against the implemented code and phase artifacts.

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Wise sync pipeline -> database cleanup | A successful external sync triggers deletion of old internal snapshot data. | Snapshot IDs, snapshot-scoped database rows, sync metadata |
| Database metadata -> operator visibility | Pruning, stale, failure, and sync status are stored in DB metadata and displayed to admins. | Operational metadata, failure summaries, stale age |
| Browser -> `/api/data-health` | Authenticated client fetches operational status. | Session cookie, stale/failure metadata |
| Session storage -> banner UI | Browser-local dismissal affects warning visibility only. | Local dismissal flag |
| Browser -> `/api/internal/sync-wise` | Authenticated admin browser can trigger an expensive Wise sync. | Session cookie, manual sync request |
| Vercel Cron -> `/api/internal/sync-wise` | Cron GET remains bearer-secret authenticated. | Authorization bearer secret |
| npm registry -> lockfile | Dependency metadata is resolved into `package-lock.json`. | Package names, versions, integrity hashes |
| Compare API -> client | Conflict data and modality labels are serialized for admin UI. | Tutor/session comparison output |
