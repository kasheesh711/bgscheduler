<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Wise sync pipeline -> database cleanup | A successful external sync triggers deletion of old internal data. |
| Database metadata -> operator visibility | Pruning results are stored in JSON metadata and displayed or inspected later. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.7-01 | Tampering | `pruneOldSnapshots` | mitigate | Protect latest 30 plus active snapshot before deleting; integration tests assert active preservation. |
| T-08.7-02 | Denial of Service | Snapshot deletes | mitigate | Manual ordered deletes avoid FK errors; pruning runs after successful promotion so cleanup failure does not block fresh data. |
| T-08.7-03 | Information Disclosure | Pruning metadata | accept | Metadata contains counts and error message only; no Wise API secrets or student data are added. |
| T-08.7-04 | Tampering | `past_session_blocks` | mitigate | Helper must not import or reference `schema.pastSessionBlocks`; tests assert row count unchanged. |
</threat_model>
