Both files share the same pattern: a `useState` seeding `nowSnapshot` at mount + a `useEffect` with `setInterval` that updates minutes every 60s. Neither re-checks `isCurrentWeek` so a tab open across midnight shows the today-indicator in yesterday's column.

Fix: track the current Bangkok date (`dateString`) inside `nowSnapshot`, and have the interval tick update it. The existing `isCurrentWeek = weekStart === getCurrentMonday()` computation is already derived from `Date.now()` in `getCurrentMonday()`, but it is only recomputed when the component re-renders — which does NOT happen on a pure interval tick unless state changes.

The fix recomputes the week-start check on each tick and, if the week rolls over (e.g., tab was open 11:59 Sunday → now 00:00 Monday), forces a state update by including the date in the snapshot.

**Current shape (in BOTH files):**
```typescript
const [nowSnapshot, setNowSnapshot] = useState(() => {
  const bkk = new Date(new Date().toLocaleString("en-US", { timeZone: "Asia/Bangkok" }));
  return { minutes: bkk.getHours() * 60 + bkk.getMinutes(), dow: bkk.getDay() };
});

const isCurrentWeek = weekStart === getCurrentMonday();

useEffect(() => {
  if (!isCurrentWeek) return;
  const tick = () => {
    const bkk = new Date(new Date().toLocaleString("en-US", { timeZone: "Asia/Bangkok" }));
    setNowSnapshot({ minutes: bkk.getHours() * 60 + bkk.getMinutes(), dow: bkk.getDay() });
  };
  const id = setInterval(tick, 60_000);
  return () => clearInterval(id);
}, [isCurrentWeek]);
```

**Change to (apply the same edit to BOTH files):**
```typescript
const [nowSnapshot, setNowSnapshot] = useState(() => {
  const bkk = new Date(new Date().toLocaleString("en-US", { timeZone: "Asia/Bangkok" }));
  return {
    minutes: bkk.getHours() * 60 + bkk.getMinutes(),
    dow: bkk.getDay(),
    dateKey: bkk.toDateString(),
  };
});

const isCurrentWeek = weekStart === getCurrentMonday();

// Tick every minute. Always runs (not gated on isCurrentWeek) so a tab left
// open across midnight re-evaluates the week by updating dateKey → triggers
// a re-render → isCurrentWeek recomputes via getCurrentMonday(). POLISH-07/M2
// fix from v1.0-MILESTONE-AUDIT.md:134.
useEffect(() => {
  const tick = () => {
    const bkk = new Date(new Date().toLocaleString("en-US", { timeZone: "Asia/Bangkok" }));
    const next = {
      minutes: bkk.getHours() * 60 + bkk.getMinutes(),
      dow: bkk.getDay(),
      dateKey: bkk.toDateString(),
    };
    setNowSnapshot((prev) =>
      prev.minutes === next.minutes && prev.dow === next.dow && prev.dateKey === next.dateKey
        ? prev
        : next,
    );
  };
  const id = setInterval(tick, 60_000);
  return () => clearInterval(id);
}, []);
```

**Notes:**
- Removing the `isCurrentWeek` gate is intentional: the interval must run even when `isCurrentWeek` is currently false so it can fire exactly once at the first tick after midnight on the week boundary, at which point `dateKey` changes, the component re-renders, and `getCurrentMonday()` returns a new value flipping `isCurrentWeek` to true. One tick per minute; negligible overhead.
- The render-site guards (`isCurrentWeek && nowSnapshot.dow === day && ...`) at calendar-grid.tsx:300 and week-overview.tsx:544 remain unchanged — they correctly no-op when `isCurrentWeek` is false.
- `bkk.toDateString()` is stable across minutes within the same day and changes at midnight, giving us a cheap day-roll detector.
