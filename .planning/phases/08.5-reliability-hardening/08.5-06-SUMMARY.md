## Decisions Made

- **Swallow-not-rethrow semantics preserved** — The cleanup catch returns void; the primary `errorMessage` (already in scope from line 519) is still the canonical SyncResult.errorSummary. Masking it with a cleanup error would be a regression.
- **`console.error` over `console.warn`/`console.log`** — Cleanup failures are operational errors that need to hit Vercel's error log stream so they show up in the existing log-filtering UI; `console.warn` would be tier-mismatched and `console.log` would not appear in error views.
- **Single log line, two embedded errors** — Easier to correlate than two separate calls. Primary error is quoted for log-search filterability; cleanup message is the second arg so its stack/object representation prints natively.
- **Log prefix `[sync-orchestrator] cleanup failed`** — Matches project convention (bracketed module name + verb phrase); the literal substring `[sync-orchestrator] cleanup failed` is the assertion target in the plan acceptance criteria for greppability.
