## Accomplishments

- Removed the timing-attackable `authHeader !== `Bearer ${cronSecret}`` string comparison
- Introduced `timingSafeEqual(received, known)` from `node:crypto` over equal-length Buffer pairs
- Added an O(1) length-equality short-circuit BEFORE `timingSafeEqual` to prevent the RangeError throw on length mismatch and to avoid leaking the secret length via the timing of the throw vs the compare
- Split missing-CRON_SECRET into a dedicated 500 ("Server misconfigured") path so deployment misconfiguration is operationally distinguishable from auth failure
- Coalesced the optional `authorization` header to `""` so `Buffer.from` cannot receive null/undefined
- Mitigated threats T-08.5-14 (timing attack on CRON_SECRET) and T-08.5-15 (length-leak via timing on length-mismatched inputs)

## Decisions Made

1. **Short-circuit form over Cloudflare's `!timingSafeEqual(a,a)` form.** Both are valid and constant-time; the short-circuit is the canonical Node-docs pattern and is more readable because the length-equality precondition is explicit. RESEARCH.md REL-07 deep-dive (lines 601-657) explicitly recommends this form.
2. **Missing-CRON_SECRET → 500, not 401.** A missing env var is a deployment misconfiguration, not an authorization failure. Splitting these paths gives operations a meaningful signal: "fix the deployment" vs "this looks like an auth probe." Per RESEARCH.md REL-07 deep-dive recommendation.
3. **Null-coalesce header to `""`.** `request.headers.get('authorization')` returns `string | null`. Passing `null` to `Buffer.from(null)` throws TypeError. The `?? ""` coalesce keeps the constant-time path branch-free and predictable.
4. **No new route-handler tests this plan.** Per the plan's `<verification>` and the Testing-strategy table in RESEARCH.md, route-handler tests for `handleSync` (3 cases: valid, invalid, wrong-length) are deferred to Phase 8.6 TCOV-03. Manual code-read + grep-based acceptance criteria suffice for REL-07.
