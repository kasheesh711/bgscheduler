## Files Created/Modified

- `src/lib/search/__tests__/index.test.ts` (created, 172 lines) — 3-case suite verifying single-build coalescing, fast cached path, and stale-cache race handling. Uses an instrumented fake `Database` that exposes a chainable thenable mimicking Drizzle's `select().from().where().limit()` API and counts `db.select` invocations as the observable side-effect.

Hoisted in-flight check moves to the top; cached-snapshot lookup + `buildIndex(db)` are wrapped in a `buildAndCheck` async closure; `setBuildingPromise(p)` is assigned synchronously before any await, immediately followed by `return p`.

## Decisions Made

- **Use `db.select` call counting instead of `vi.spyOn(indexModule, "buildIndex")`** — ESM live bindings mean `ensureIndex`'s internal `buildIndex(db)` call goes through the local function reference, not the namespace property, so a spy on the export cannot intercept it. The plan explicitly authorized this fallback ("If the spy approach is not feasible due to module internals, fall back to **counting actual DB calls** instead"). This keeps the test pure-Vitest, no module-mocking incantations needed.
- **Wrap BOTH the cached-snapshot lookup AND the buildIndex call inside `buildAndCheck`** — the original race wasn't only between two callers about to kick off `buildIndex`; it was specifically between two callers BOTH doing the cached-snapshot lookup before either reached the build kickoff. The fix needs to coalesce that earlier work too. By moving the cached check inside the singleton-protected closure, the second caller's fast-path return at the top of `ensureIndex` (`if (inFlight) return inFlight;`) avoids the duplicate cached-check select.
- **Hoist the in-flight check to the top of `ensureIndex`** — strictly stronger than only checking inside the build-kickoff branch. A concurrent caller arriving during another caller's build now exits before doing ANY work (no cached-check, no buildIndex).

Test 1's purpose is to lock in the single-build invariant, not to reproduce the race. Test 3 (concurrent stale-cache callers) IS the live-race reproducer and DOES fail RED against the buggy code.
