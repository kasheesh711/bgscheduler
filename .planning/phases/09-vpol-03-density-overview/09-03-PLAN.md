<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser-rendered density UI -> admin decision-making | Visual density must not be mistaken for proven availability. |
| Assistive technology -> density labels | Screen-reader users receive the same booked-hours/session-count meaning as sighted users. |
| Verification artifact -> repo history | Evidence must not include secrets, cookies, tokens, or private logs. |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|-----------|----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | Medium | Manual a11y verification | mitigate | VoiceOver check confirms React labels announce dynamic strings without HTML injection APIs. |
| T-09-02 | Elevation of Privilege | Medium | Segment activation | mitigate | Browser check confirms activation only opens existing day view and does not write, fetch, or schedule. |
| T-09-03 | Information Integrity | Medium | Visual/copy verification | mitigate | Manual copy check blocks availability/capacity/utilization/free/recommended claims. |
