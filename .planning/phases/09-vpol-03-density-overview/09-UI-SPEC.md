# Phase 09 - UI Design Contract

> Visual and interaction contract for VPOL-03 Density Overview. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui v4 detected by `components.json` and `npx shadcn info` |
| Preset | `base-nova` |
| Component library | shadcn/ui wrappers in `src/components/ui`; Base UI primitives already present |
| Icon library | lucide, but no new icon is required for this phase |
| Font | Inter for body/heading; JetBrains Mono only for existing mono surfaces |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Segment gaps, row gaps, inner segment padding |
| sm | 8px | Row label-to-segment gap, compact vertical spacing |
| md | 16px | Default compare-panel spacing; do not add extra md padding around density |
| lg | 24px | Existing sticky legend height reference; density must not consume this as per-row padding |
| xl | 32px | Density segment height and compact row target |
| 2xl | 48px | Reserved for surrounding calendar grid scale, not density overview |
| 3xl | 64px | Page-level spacing only; not used in density overview |

Exceptions: use a fixed 96px tutor-label column and 3px tutor-color left rail on nonzero segments. The 3px rail is a visual identity marker, not layout spacing. No 44px touch-target exception is required because the app remains desktop-admin focused and the density action is a compact keyboard-accessible shortcut.

---

## Typography

Use exactly these four sizes and only weights 400 and 600 in the density overview.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Micro | 10px | 400 | 1.2 |
| Body | 12px | 400 | 1.5 |
| Label | 11px | 600 | 1.2 |
| Heading | 14px | 600 | 1.2 |

Usage: tutor row names use 11px/600 and truncate. Row summaries use 10px/400. Segment hour labels use 10px/600 only when booked minutes are greater than zero. No display-scale text is used in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / `oklch(0.985 0.005 85)` light, `oklch(0.16 0.008 250)` dark | Compare panel surface and density strip background |
| Secondary (30%) | `var(--muted)`, `var(--secondary)`, `var(--border)` | Segment empty beds, row separators, secondary labels, hover surface |
| Accent (10%) | Existing `tutorChips` colors from `TUTOR_COLORS`: `#3b82f6`, `#e67e22`, `#7c3aed` | Tutor identity only: row dot/name, nonzero segment fill, 3px segment rail |
| Destructive | `var(--destructive)` / existing `var(--conflict)` | Inherited compare error/conflict surfaces only; density adds no destructive action |

Accent reserved for: tutor row identity dot, tutor row name, density segment fill, density segment left rail, and selected-day/focus treatment via existing `--primary`/`--ring`. Do not use `bg-available`, `bg-blocked`, `bg-free-slot`, or a new density palette in this component. Density must not imply availability.

## Phase Surface Contract

| Surface | Contract |
|---------|----------|
| Component | Add `DensityOverview` as a small client component, preferably `src/components/compare/density-overview.tsx`, with an exported pure aggregation helper for tests. |
| Placement | Render in `ComparePanel` after the day tabs and before the calendar body wrapper. It must remain visible for both week view and day drill-down while `compareResponse` is loaded. |
| Shape | Shape B only: one stacked row per selected tutor, seven day segments ordered Monday-Sunday. Do not re-open aggregate bar or heatmap shapes except in the design-review artifact. |
| Container | Full-width, unframed strip with `flex-shrink-0`, subtle top/bottom or bottom border, and compact `py-1`. Do not style it as a card and do not place it inside another card. |
| Row Geometry | `grid-cols-[96px_1fr]`, 8px column gap, 4px row gap, max three rows. Total height at three tutors should stay at or below 120px. |
| Segment Geometry | Seven equal columns, 4px gap, 32px height, 4px radius, `border-border/60`, `bg-muted/30`. Nonzero booked time gets a tutor-colored fill width plus 3px left rail. |
| Visible Labels | Row label shows truncated tutor name and `{weeklyHoursBooked}h booked`. Segment visible label shows compact booked hours such as `1h` or `1.5h`; zero-hour segments may be visually blank but must keep text equivalents. |
| Selected Day | When `activeDay` is a weekday, all matching weekday segments use `aria-current="date"` and a primary/ring outline or border treatment. When `activeDay === null`, no day segment is current. |

## Data And Density Contract

| Item | Contract |
|------|----------|
| Source | Derive entirely from existing `CompareResponse.tutors[].sessions[]`, `availabilityWindows`, and `weeklyHoursBooked`. |
| Memoization | Compute rows with `useMemo` keyed by `compareResponse.tutors` and `compareTutors`/`tutorChips`; do not depend on unrelated compare panel state. |
| Booked Minutes | Per tutor/day, sum `Math.max(0, endMinute - startMinute)` for sessions whose `weekday` matches the day. |
| Session Count | Count sessions in the same tutor/day bucket; expose count in aria/title labels. |
| Fill Formula | Normalize visible fill by raw booked minutes against the maximum booked tutor/day cell in the visible 1-3 tutor x 7 day set. Use a minimum denominator of 60 minutes and a minimum visible fill of 12% only for nonzero booked time. |
| Zero Data | If all booked minutes are zero, every fill ratio is zero and rows still show `0h booked`; never divide by zero and never render a full-looking cell. |
| Availability | Availability windows may be read as guard/context data, but do not display utilization percentages or capacity claims. |
| Cache/API | No new API route, DB query, SearchIndex field, schema change, localStorage key, or `CACHE_VERSION` bump. Keep `CACHE_VERSION = "v2"` unless a cached response shape change is explicitly introduced and documented. |

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Segment click | Native `<button type="button">`; click calls existing day drill-down path by setting `activeDay` to that weekday. |
| Keyboard | Standard button keyboard behavior only: Tab focus, Enter/Space activate. Do not implement roving focus or arrow-key grid navigation. |
| Hover | Use a subtle muted/primary border treatment only. No tooltip dependency is required; native `title` is allowed as a secondary affordance. |
| Focus | Use `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-1` or equivalent existing focus-ring treatment. |
| Motion | Static render. No animated fill, pulse, shimmer, delayed reveal, or transition that changes density state. `prefers-reduced-motion: reduce` must have no density-specific animation to suppress. |
| Forbidden Actions | Do not jump to an hour, highlight individual sessions, open a scheduling flow, show shared-free-slot overlays, show utilization percentages, or mutate tutor selection. |

## Accessibility Contract

| Element | Contract |
|---------|----------|
| Region | Wrap the overview in a `section` with `aria-label="Visible week booking density"`. |
| Row | Keep tutor name and weekly booked-hours summary visible, not screen-reader only. |
| Segment Label | Each segment button must expose: `{Day}: {Tutor}, {hours} booked, {sessionCount} session(s). Open day view.` through `aria-label`; matching `title` is allowed. |
| Non-Color Encoding | Density is encoded by visible hours text, fill width, and row summary. Tutor color identifies the tutor only. |
| Text Overflow | Tutor names truncate in the 96px label column; segment labels must never overlap adjacent segments. Hide visible segment text before allowing overflow. |
| Screen Reader Scope | VoiceOver spot-check must confirm day, tutor, booked hours, session count, and action are announced. NVDA remains deferred per v1.2 backlog. |
