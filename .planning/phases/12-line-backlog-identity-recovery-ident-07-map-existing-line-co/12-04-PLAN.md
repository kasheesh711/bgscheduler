  <how-to-verify>
CRITICAL — READ THIS BEFORE PROCEEDING:

The local `.env.local` contains `LINE_CHANNEL_ACCESS_TOKEN` for the **"BeGifted Testing"** channel — a different LINE OA with different followers. You MUST use the production token for this dry-run to see the real `@begifted` followers.

**Step 1: Pull production env**
```bash
vercel env pull --environment=production .env.production.local
```
This writes a `.env.production.local` file with the production LINE token.

**Step 2: Run the dry-run via the production endpoint**
Log in to https://bgscheduler.vercel.app as an admin user (use your Google account that is on the allowlist), then open DevTools → Application → Cookies → copy the `next-auth.session-token` cookie value.

Then:
```bash
curl -s -X POST \
  "https://bgscheduler.vercel.app/api/line/contacts/followers-reanchor?dryRun=true" \
  -H "Cookie: next-auth.session-token=<your-session-token>" \
  | jq '.backlog'
```

OR if the endpoint is accessible via the admin UI ("Re-anchor followers" button), click it with `?dryRun=true` appended to the URL in the button's handler.

**Step 3: Inspect the response**
The `backlog` field in the response should contain:
```json
{
  "contactsScanned": <number>,
  "targetsCount": ~662,
  "matchedCount": ~323,
  "insertedCount": 0,
  "dryRun": true,
  "dryRunMatches": [
    {
      "lineUserId": "U...",
      "displayName": "<follower display name>",
      "matchedStudentKey": "<studentKey>",
      "studentName": "<student name from resolver row>",
      "lineChatUrl": "https://chat.line.biz/...",
      "confidence": "high",
      "tokens": ["<matched token>"]
    },
    ...
  ]
}
```

Expected: ~229 entries with `confidence: "high"` and ~94 with `confidence: "ambiguous"`.

**Step 4: Spot-check ≥10 entries**
For each spot-check entry:
- `lineChatUrl` must start with `https://chat.line.biz/` (not null, not a placeholder)
- `studentName` should match a real student you recognize from BeGifted
- `displayName` should look like the parent/student's LINE display name (not a dotted code)
- Open 2-3 chat URLs in the browser — confirm they open real LINE OA chats

**Step 5: Compare against UAT CSV**
If `/tmp/line-identity-matches.csv` is still on your machine from Phase 11 UAT:
```bash
head -20 /tmp/line-identity-matches.csv
```
Cross-check 5-10 names between the CSV and the dry-run output.

**Step 6: Check for unexpected entries**
- Are there any matches that look wrong? (wrong student name for a follower you recognize)
- Are high-confidence matches (confidence: "high") sensible?
- Are ambiguous matches (confidence: "ambiguous") reasonably multi-student?

Type "approved" below if the dry-run output looks correct.
Type "issues: <description>" if you see unexpected results that need investigation before live write.
