In `src/components/compare/compare-panel.tsx`, replace the three static imports with `next/dynamic` calls at MODULE SCOPE (top of file, not inside the component function -- this is critical for the bundler per Next.js 16 lazy-loading docs).

**Remove these static imports:**
```typescript
// DELETE these three lines:
import { CalendarGrid } from "@/components/compare/calendar-grid";
import { WeekOverview } from "@/components/compare/week-overview";
import { DiscoveryPanel } from "@/components/compare/discovery-panel";
```

**Add dynamic imports at module scope (after the other imports):**
```typescript
import dynamic from "next/dynamic";
import { CalendarSkeleton } from "@/components/skeletons/calendar-skeleton";

const WeekOverview = dynamic(
  () => import("@/components/compare/week-overview").then((mod) => mod.WeekOverview),
  { loading: () => <CalendarSkeleton /> }
);

const CalendarGrid = dynamic(
  () => import("@/components/compare/calendar-grid").then((mod) => mod.CalendarGrid),
  { loading: () => <CalendarSkeleton /> }
);

const DiscoveryPanel = dynamic(
  () => import("@/components/compare/discovery-panel").then((mod) => mod.DiscoveryPanel),
  { loading: () => null }
);
```

Key points:
- Use `.then((mod) => mod.ComponentName)` pattern for named exports (all three components use named exports, not default exports)
- WeekOverview and CalendarGrid use `<CalendarSkeleton />` as loading fallback (per D-07)
- DiscoveryPanel uses `null` as loading fallback since it renders inside a dialog/modal and a brief flash is acceptable
- The `dynamic()` calls MUST be at module scope (outside the component function)
- Keep `"use client"` directive at top of file
- All other imports and the component body remain unchanged -- the dynamic imports produce the same component references used in JSX
