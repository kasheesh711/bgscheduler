const searchParams = useSearchParams();

SearchWorkspace does NOT need `loading` state. SearchForm owns loading internally (the button text changes to "Searching..."). SearchResults shows empty state via `!response` check.

**Init effects** (from page.tsx lines 172-188):
```typescript
// Fetch filter options
useEffect(() => {
  fetch("/api/filters")
    .then((res) => (res.ok ? res.json() : null))
    .then((data) => { if (data) setFilterOptions(data); })
    .catch(() => {});
}, []);

// Handle ?tutors= deep link
useEffect(() => {
  const tutorIds = searchParams.get("tutors")?.split(",").filter(Boolean) ?? [];
  if (tutorIds.length > 0) {
    compare.fetchCompare(tutorIds, compare.weekStart);
  }
}, []); // eslint-disable-line react-hooks/exhaustive-deps
```

**Communication wiring** (per D-06):
```typescript
const handleSearchResponse = useCallback((data: RangeSearchResponse, context: SearchContext) => {
  setResponse(data);
  setSearchContext(context);
  setError(null);
}, []);

const handleCompareSelected = useCallback((ids: string[]) => {
  compare.tutorCache.current.clear();
  compare.fetchCompare(ids, compare.weekStart);
}, [compare]);
```
