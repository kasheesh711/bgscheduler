## Critical Anti-Patterns (do NOT repeat these)
- **Don't spawn a new worktree executor before pruning stale worktrees** — sequential dispatch doesn't help if the `.git/config.lock` is already held by an orphaned `locked` file. Structural fix: run `git worktree prune` (and, if needed, `rm -rf .git/worktrees/*/locked` then prune again) BEFORE the Wave 4 `Task(isolation="worktree")` call.

Orchestrator must commit it manually before running the merge (as was done for 06-03 and 06-04 in this session).
